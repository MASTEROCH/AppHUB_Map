# AppHub.Studio — карта экосистемы

Интерактивная карта продуктов AppHub: одна петля трафика **L1 → L2 → L3 → L1**
(пользователи → агрегаторы → бизнесы → снова пользователи) и 4 источника трафика по углам.

## Файлы

| Файл | Для кого | Что умеет |
|------|----------|-----------|
| **`index.html`** | 🌐 публичная витрина (клиенты) | просмотр, демо-тур, фильтры, поиск, статусы, ссылки на продукты. **Без редактирования.** |
| **`AppHub-карта-экосистемы.html`** | 🔒 внутренняя (команда) | всё то же + редактор: описания, скриншоты, ссылки, статусы, создание/перемещение узлов и связей, экспорт/импорт JSON. |

Оба файла **самодостаточные** — просто открой двойным кликом в браузере (Chrome / Safari). Сервер не нужен.

## Как пользоваться

- **Клик по узлу** — выезжает карточка с описанием, статусом и живой ссылкой на продукт.
- **▶ Демо-тур** — интерактивный онбординг за минуту (на витрине запускается сам при первом визите).
- **Фильтры** (Все / Живые / В разработке / Концепты) и **поиск** — сверху.
- **Зум**: колесо мыши, пинч, кнопки. **Перетаскивание** — двигает карту.
- **Глубокая ссылка**: `index.html#dine` открывает карту сразу на конкретном продукте — удобно отправлять клиенту.

## Статусы продуктов
- 🟢 **Живой** — в проде (рабочая ссылка)
- 🟡 **В разработке**
- ⚪ **Концепт**

## Как менять данные (для команды)

Единый источник правды — папка **`src/`**:
- `src/data.js` — все узлы, связи, статусы, координаты
- `src/style.css` — стили (Liquid Glass)
- `src/app.js` — логика
- `src/public.html`, `src/internal.html` — шаблоны

После правки в `src/` пересобери корневые файлы:

```bash
python3 src/build.py
```

Это вшивает CSS/JS/данные в самодостаточные `index.html` и `AppHub-карта-экосистемы.html`.

Можно и без кода: открыть внутреннюю версию → **Редактор** → поправить → **Экспорт JSON**, и передать JSON для вшивания в `data.js`.
